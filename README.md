# Shortest Path Finder: Система пошуку найкоротших шляхів

## Огляд Проєкту

Shortest Path Finder — це інтерактивний веб-застосунок, розроблений для демонстрації та порівняльного аналізу ефективності двох фундаментально різних алгоритмів пошуку найкоротшого шляху: класичного **Алгоритму Дейкстри** та метаевристичного **Мурашиного алгоритму (Ant Colony Optimization, ACO)**.

Система використовує **MongoDB** як гнучке сховище для постійного зберігання складних графових структур та результатів обчислень.

## Ключові Можливості

* **Інтерактивний редактор графів:** Створення, редагування, збереження та візуалізація власних графів. Можлива генерація випадкових графів (до 50 вершин) або використання передвстановленого графа міст України.
* **Два алгоритми:** Реалізовано запуск Дейкстри (точний мінімум) та ACO (евристика).
* **Порівняльний Аналіз:** Пряме порівняння часу виконання та знайдених відстаней між ACO та Дейкстрою.
* **Налаштування ACO:** Користувач може тонко налаштовувати параметри ACO (кількість мурах, Alpha, Beta, ітерації) для вивчення його збіжності та впливу на ефективність.
* **Зберігання Даних:** Усі графи та результати пошуку зберігаються у базі даних MongoDB.

## Архітектура та Алгоритми

Проєкт реалізовано з використанням трирівневої архітектури, де Flask виступає як рівень застосунку (Application Layer).

### 1. Рівень Даних (MongoDB)

* **MongoDB:** Використовується як основне сховище NoSQL. Це дозволяє зберігати складні об'єкти (граф, який містить масиви вузлів та ребер) у природному для них форматі (JSON-документи).
* **Структура:** Дані організовані в колекції `graphs` та `search_results`.

### 2. Рівень Обчислень (Python)

* **Ant Colony Optimization (ACO):** Метаевристичний алгоритм, що імітує поведінку мурах. Він використовується для знаходження квазі-оптимальних шляхів, особливо ефективний для великих та складних задач оптимізації.
* **Алгоритм Дейкстри:** Класичний та точний алгоритм для пошуку найкоротшого шляху в зваженому графі з невід'ємними вагами. Слугує еталоном для перевірки точності ACO.
* **NumPy:** Використовується для ефективної роботи з матрицями суміжності та феромонів.

### 3. Рівень Інтерфейсу (Frontend)

* **Flask / Jinja2:** Генерує динамічні HTML-сторінки.
* **Vis.js:** Бібліотека для інтерактивної візуалізації графів у реальному часі.
* **Chart.js:** Використовується для відображення графіків збіжності ACO.

## Технологічний Стек

* **Backend:** Python 3.10+, Flask, PyMongo, NumPy.
* **Database:** MongoDB.
* **Frontend:** HTML5, CSS3, Bootstrap 5, Vis.js, Chart.js.

## Запуск Проєкту

1. Вимоги
Python 3.10 або новіший.

Встановлений сервер MongoDB (локально або доступ до віддаленого URI).

2. Встановлення
Клонуйте репозиторій та встановіть залежності:

Bash

git clone [Ваш_URL_репозиторію]
cd kyrsova
python -m venv venv
# Активація середовища:
# Windows: venv\Scripts\activate
# Linux/macOS: source venv/bin/activate

# Встановлення залежностей
pip install -r requirements.txt
3. Налаштування Середовища (.env)
Створіть файл .env у корені проєкту та вкажіть налаштування MongoDB. Переконайтеся, що MONGODB_URI вказує на робочий сервер:

# MongoDB Configuration
MONGODB_URI=mongodb://localhost:27017/
MONGODB_DATABASE=shortest_path_db

# Flask Configuration
FLASK_APP=app.py
FLASK_ENV=development
SECRET_KEY=your-secret-key-change-in-production
4. Запуск Сервера
Запустіть Flask-додаток:
python app.py
Сервер буде доступний за адресою: http://127.0.0.1:5000

Інструкція з Використання
Створення Графа: Перейдіть на сторінку /graph/editor. Створіть граф вручну або скористайтеся кнопками "Випадковий граф" чи "Граф міст". Після створення натисніть "Зберегти".

Пошук: Перейдіть на сторінку /search.

Оберіть щойно збережений граф у випадаючому списку.

Виберіть Початкову та Кінцеву вершини.

Виберіть алгоритм (рекомендується "Порівняти обидва" для аналізу).

Натисніть "Знайти шлях".

Аналіз: Перегляньте таблицю порівняння, яка демонструє різницю у відстані та часі виконання між Дейкстрою (точний мінімум) та ACO (евристика).

Тестування
Для запуску юніт- та інтеграційних тестів, які перевіряють логіку алгоритмів, коректність роботи з базою даних та стабільність API:

# Переконайтеся, що venv активовано
python -m unittest discover tests
